Upgrade the “Interview Ready AI” app to use a free AI API for personalized resume analysis and report generation.

Current app:
- User uploads resume as PDF, DOCX, PNG, or JPG
- App extracts resume text
- App calculates an Interview Readiness Score using rule-based logic
- App shows category-wise score dashboard

Required upgrade:
Add AI-powered feedback generation using the Gemini API.

Important:
Do not let the AI randomly generate the score.
The score should still come from the app’s rule-based scoring logic.
The AI should only generate personalized feedback, improvement suggestions, recruiter first impression, and 7-day preparation plan based on the extracted resume text and calculated scores.

Use this flow:
1. User uploads resume or pastes resume text.
2. App extracts readable resume text.
3. App calculates category scores:
   - Resume Quality: 25 points
   - Technical Skills Match: 25 points
   - Project/Portfolio Strength: 20 points
   - Communication & Presentation: 15 points
   - Job Role Fit: 15 points
4. App sends the resume text, target role, and calculated category scores to Gemini API.
5. Gemini returns a structured JSON report.
6. App displays the final report in the dashboard.

Use Gemini API:
- Use @google/generative-ai or the latest official Gemini JavaScript SDK
- Store the API key in an environment variable:
  VITE_GEMINI_API_KEY
- Never hardcode the API key in the frontend code if possible.
- If using a frontend-only MVP, clearly separate the API key config and include comments that production should use a backend proxy.

Create an AI report generation function called generateAIReport.

AI prompt rules:
- Keep feedback specific to the target role.
- Do not be generic.
- Do not invent experience, projects, companies, certifications, or achievements that are not present in the resume.
- If information is missing, say it is missing.
- Keep suggestions beginner-friendly for students.
- Make the feedback actionable.
- Do not include markdown in the JSON.
- Return only valid JSON.

Add fallback behavior:
If the Gemini API fails, show the existing rule-based report instead and display:
“AI feedback is temporarily unavailable, but your rule-based readiness report is ready.”
